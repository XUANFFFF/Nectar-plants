# Prompt 记录

本文档记录本原型开发过程中使用 AI 编程工具的关键协作节点。内容为整理版，重点保留每次 Prompt 对产品方向或功能实现产生的影响。

## 1. 数据理解

Prompt：

```text
读一下这份 Excel，帮你总结字段和能做的产品方向
```

产出：

- 识别出数据包含物种、观察者、地点、经纬度、图片、访花关联、审核状态等信息。
- 初步判断可做方向包括数据看板、植物图鉴、地图探索、志愿者贡献展示等。

## 2. 产品方向调整

Prompt：

```text
数据看板感觉偏后台审核使用，我想做一个面向用户和参与者的，动态展示数据、同时让参与者感受到自己提交数据的价值
```

产出：

- 产品定位从“后台审核看板”调整为“参与者共创体验平台”。
- 形成三个主方向：共创地图、植物图鉴、个人贡献。
- 强调数据不只是被管理，也要被参与者看见和理解。

## 3. 页面方向选择

Prompt：

```text
2 + 3 为主，1 为辅
```

产出：

- 确认以共创任务和生态地图为主，数据总览为辅助。
- 页面结构调整为首页、地图、图鉴、动态、个人贡献。

## 4. 场域交互

Prompt：

```text
要增加一个交互功能，如果用户到达某个花园，弹出这个场域的相关信息，以便用户有针对性的探索
```

产出：

- 增加场域信息卡思路。
- 明确地图不只是点位展示，还应该帮助用户理解“这个地方值得看什么”。

## 5. 使用真实图片

Prompt：

```text
associatedMedia 列中有图片链接，使用这些图片
```

产出：

- 从 Excel 的 `associatedMedia` 字段解析图片链接。
- 将可访问的图片缓存到 `public/media/`。
- 植物图鉴和动态卡片优先使用真实观察图片。

## 6. 图鉴卡片统一

Prompt：

```text
统一下不同尺寸的图片在图鉴页的展示大小，确保图鉴卡片尺寸统一
```

产出：

- 固定植物卡片图片区域高度。
- 图片使用裁切适配，避免不同图片比例破坏布局。

## 7. 地图位置修复

Prompt：

```text
现在数据和地图位置对不上
```

产出：

- 检查城市边界与观察点的坐标映射。
- 统一地图投影逻辑，使经纬度点位和广东城市边界对齐。

## 8. 地图选择优化

Prompt：

```text
数据太多选不到城市了
```

产出：

- 将观察点从可点击按钮层调整为视觉热度层。
- 提高城市边界点击层级，让用户可以直接选择城市。
- 缩小并弱化点位，避免省级地图被点位覆盖。

## 9. 提交材料整理

Prompt：

```text
根据文档要求整理一份供提交的材料
```

产出：

- 生成提交版 `README.md`。
- 补充 `process/` 过程记录。
- 补充 `data/field_notes.md` 字段说明。

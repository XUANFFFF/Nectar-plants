# 问题与修复记录

## 1. 原型定位偏后台

现象：

最初容易做成数据审核看板，虽然信息完整，但参与者不容易感受到自己的数据价值。

处理：

调整产品目标为面向用户和参与者的共创平台，增加地图、图鉴、伙伴动态、我的贡献等页面。

## 2. 地图点位与城市边界不匹配

现象：

观察点位显示在地图上时，与广东城市边界位置不一致。

处理：

统一城市边界和观察点位的坐标投影方法，使用同一套经纬度映射逻辑。

## 3. 植物图鉴缺少真实图片

现象：

Excel 中有 `associatedMedia` 图片链接，但植物图鉴仍然显示占位图。

处理：

解析图片链接并缓存到 `public/media/`，图鉴卡片和详情弹窗优先使用真实图片。

## 4. 图鉴卡片高度不一致

现象：

不同图片比例导致植物卡片大小不一致，视觉上不稳定。

处理：

固定图片区域高度，图片使用统一裁切适配，卡片采用统一网格高度。

## 5. 地图点位过多影响城市选择

现象：

省级地图上观察点较多，点位按钮覆盖城市边界，导致用户难以点击城市。

处理：

将点位改为不拦截点击的视觉热度层，城市边界作为主要交互层。点位缩小并降低透明度，保留数据分布感。

## 6. 静态原型缺少后端能力

现象：

当前版本使用本地 JSON 和静态图片，不支持在线提交、实时审核或多人协作。

处理：

在黑客松阶段接受静态原型约束，优先保证可展示、可点击、可说明。后续可接入后端或在线表单。
